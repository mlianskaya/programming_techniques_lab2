Detailed Description

Класс хэш-таблицы для быстрого поиска по ключу.

  • Размер таблицы фиксирован и задаётся в конструкторе (рекомендуется простое число).
  • Коллизии разрешаются методом цепочек: каждая корзина — это std::unordered_map.
  • Внутри корзины ключи уникальны, а для каждого ключа хранится вектор всех значений.
  • Дубликаты (одинаковые ФИО) не создают коллизий, а добавляются в существующий вектор.
  • Ведётся счётчик коллизий (сколько раз при вставке нового уникального ключа корзина уже была непуста).

Parameters
keyИскомое ФИО.
Returns
Вектор копий найденных объектов. Если ключ не найден, возвращается пустой вектор.
  • Вычисляется хэш, находится корзина.
  • В корзине выполняется поиск по ключу (за O(1) в среднем).
  • Если ключ найден, возвращается вектор значений; иначе – пустой вектор.
  • Сложность в среднем O(1), в худшем O(k), где k – число различных ключей в корзине.
